在人工智能领域，智能体被定义为任何能够通过**传感器（Sensors）**感知其所处**环境（Environment）**，并**自主**地通过**执行器（Actuators）**采取**行动（Action）**以达成特定目标的实体。

大脑：模型；身体：工具

# Agent 能力层次笔记

## 1. 核心理解

智能体能力可以理解为一条逐步增强的连续谱：

> **回答 → 路由 → 调工具 → 自主循环 → 调度其他智能体**

能力越强，LLM 对程序执行流程的控制权越大。

---

## 2. 五个能力层次

| 层级 | 核心能力         | 直观理解                         |
| ---- | ---------------- | -------------------------------- |
| ☆☆☆  | 只输出结果       | LLM 负责回答，不控制程序流程     |
| ★☆☆  | 决定走哪条路     | LLM 进行条件判断和流程路由       |
| ★★☆  | 决定调用什么工具 | LLM 自主选择函数、API、数据库等  |
| ★★★  | 决定下一步做什么 | LLM 在循环中持续观察、决策和行动 |
| ★★★+ | 调用其他智能体   | 一个 Agent 调度多个专业 Agent    |

---

## 3. 各层级解释

### 3.1 ☆☆☆ 简单处理器

固定流程：

```text
输入
→ LLM处理
→ 输出
```

例如文献抽取：

```text
输入论文
→ LLM抽取字段
→ 输出JSON
```

特点：

* 程序流程固定
* LLM 只是处理输入并返回结果
* 严格来说，更接近“LLM应用”而非完整 Agent

---

### 3.2 ★☆☆ 路由

LLM 开始决定任务应该进入哪条流程：

```text
输入
→ LLM判断任务类型
→ 选择对应流程
```

例如：

```text
如果是实验论文
→ 实验数据抽取流程

如果是综述论文
→ 综述处理流程
```

核心变化：

> LLM 从“回答问题”升级为“决定走哪条路”。

---

### 3.3 ★★☆ 工具调用

LLM 可以自主选择和调用外部工具：

```text
LLM判断需求
→ 选择工具
→ 传入参数
→ 获取结果
```

例如：

```text
发现表格
→ 调用表格解析工具

发现单位不统一
→ 调用单位标准化函数

需要查询DOI
→ 调用文献API
```

核心变化：

> LLM 不只是选择流程，而是能够主动执行操作。

---

### 3.4 ★★★ 多步智能体

Agent 可以持续执行多轮：

```text
观察
→ 判断
→ 行动
→ 获取结果
→ 再判断
→ 再行动
```

例如：

```text
读取论文
→ 判断缺失字段
→ 调用抽取工具
→ 检查结果
→ 发现字段缺失
→ 再次搜索
→ 重新抽取
→ 输出最终结果
```

典型结构：

```text
while 未完成:
    观察当前状态
    决定下一步
    执行动作
    检查结果
```

核心变化：

> LLM 开始控制“任务如何持续推进”。

这通常更接近真正意义上的 **Agent Loop**。

---

### 3.5 ★★★+ 多智能体系统

一个 Agent 可以调用其他专业 Agent：

```text
主Agent
├── 文本抽取Agent
├── 表格抽取Agent
├── 单位标准化Agent
└── 结果验证Agent
```

例如文献抽取系统：

```text
Coordinator Agent
    ↓
判断任务类型
    ↓
├─ Text Extraction Agent
├─ Table Extraction Agent
├─ Figure Extraction Agent
└─ Validation Agent
```

核心变化：

> 从“自己完成任务”升级为“组织其他智能体协作完成任务”。

---

## 4. 最重要的能力递进

可以把整个 Agent 能力演化压缩成：

```text
Level 0：回答
    ↓
Level 1：路由
    ↓
Level 2：调用工具
    ↓
Level 3：自主循环
    ↓
Level 4：调度其他Agent
```

对应控制权变化：

```text
只控制输出
→ 控制分支
→ 控制动作
→ 控制任务过程
→ 控制智能体协作
```

---

## 5. 一句话总结

> **Agent 的本质不是“用了 LLM”，而是 LLM 对程序执行过程拥有多大的决策权。**

从简单到复杂：

> **回答 → 路由 → 工具调用 → 自主循环 → 多 Agent 协作**
